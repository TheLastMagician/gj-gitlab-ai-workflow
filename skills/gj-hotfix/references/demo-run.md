# Demo Run Reference

The first run did not need a production hotfix, but it clarified boundaries.

Use hotfix only for:

- P0/P1 production incidents.
- security risk.
- release blockers.
- data damage.
- core workflow unavailable.

Even when moving fast, require:

- minimal human review.
- release validation.
- rollback.
- post-fix root cause.
- tests and context updates.
